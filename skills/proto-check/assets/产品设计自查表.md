# 产品设计自查表

> 来源:《产品设计自查表.xlsx》(自查项 sheet)67 条 + `prd2prototype` 沉淀的原型规范约定 21 条,2026-06 固化。共 88 条,按功能类型分组,编号规则 `Z-<类别>-<序号>`。
>
> 末尾「原型规范类」四组(标记/版本/首页/原型说明)来源是本团队原型铁律(非 xlsx),与前面通用条目分开维护。
>
> **原型判定**(本 skill 约定,指对 HTML 原型 + 原型说明(proto-note)的检查口径):
> - **界面**:原型界面上可直接看出(必填标识、控件类型、列表操作等)
> - **说明**:界面看不出,但原型说明/字段规格表里应写明(长度、校验规则、判重逻辑等)
> - **PRD**:纯需求/后端/数据层面,原型查不出 → 列入「需人工确认」清单,不强行判
>
> 标注【固化】的条目带研发已固化的默认值,原型未特别说明时按默认值判「符合」。

| 编号 | 检查要素 | 原型判定 | 备注 |
